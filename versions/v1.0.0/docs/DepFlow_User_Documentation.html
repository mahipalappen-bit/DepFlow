Status Management

NOT STARTED

Not Started

Newly created dependencies waiting to begin work. Default status for all new items.

IN PROGRESS

In Progress

Dependencies currently being worked on by the assigned team. Active development phase.

BLOCKED

Blocked

Dependencies stuck due to external factors, waiting for resolution or clarification.

COMPLETED

Completed

Successfully finished dependencies. Move to this status only when fully complete.

Email Delivery & Troubleshooting

‚ö†Ô∏è

Not Receiving Emails?

  1. Check Spam/Junk Folder

    Automated emails may be filtered by email providers. Look for emails from "DepFlow" or with "Dependency" in the subject.

  2. Verify Email Address

    Confirm your email address is correctly configured in the team owner mappings. Contact your administrator if changes are needed.

  3. Check Email Server

    Corporate email servers may have restrictions on automated emails. Contact your IT department if emails are being blocked.

  4. Test Email Function

    Administrators can test email delivery using the API endpoint:

    curl -X POST http://localhost:3000/api/test-email \
      -H "Content-Type: application/json" \
      -d '{"team": "Your Team Name"}'

Email Best Practices

Managing Email Notifications

  • Team Lead Responsibility: Team owners should monitor for new assignments
  • Quick Response: Acknowledge new dependencies promptly
  • Status Updates: Keep dependency status current to minimize notifications
  • Email Rules: Set up email rules to organize DepFlow notifications
  • Distribution Lists: Consider using team distribution lists instead of individual emails

Create Dependencies

Endpoint: POST /api/dependencies

Purpose: Create new dependency with email notification

curl -X POST http://localhost:3000/api/dependencies \
  -H "Content-Type: application/json" \
  -d '{ "name": "React Auth Component", "description": "User authentication component", "team": "Data Engineering", "priority": "HIGH", "createdBy": "api@company.com" }'

Update Dependencies

Endpoint: PUT /api/dependencies/:id

Purpose: Update existing dependency details

curl -X PUT http://localhost:3000/api/dependencies/1 \
  -H "Content-Type: application/json" \
  -d '{ "name": "Updated Component", "team": "Analytics", "status": "IN PROGRESS", "originalTeam": "Data Engineering" }'

JavaScript/Node.js Integration

// Example: Create dependency from CI/CD pipeline

Troubleshooting Guide

Common Issues & Solutions

Login Problems:

  • Invalid Credentials: Verify username/password spelling and case
  • Page Won't Load: Check internet connection and try different browser
  • Session Expired: Log out completely and log back in

Dependency Issues:

  • Can't Edit: Only creators and admins can edit dependencies
  • Save Fails: Check all required fields are completed
  • Missing Dependencies: Check active filters and clear if necessary

Email Problems:

  • Not Receiving: Check spam folder and verify email address
  • Wrong Recipient: Contact admin to update team owner mappings
